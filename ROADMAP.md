# NullWave Roadmap

## Vision
A music organizer with ability to download and play tracks from YouTube, Spotify, SoundCloud, and local files. Organizer-first, player second.

---

## Phase 1 — Core Organizer (current focus)
- [x] Add tracks by URL or local file
- [x] Library with search, sort, favorites
- [x] Playlists
- [x] Tags and notes on tracks
- [x] Export to JSON/CSV
- [ ] Remove Instagram source (cleanup)
- [ ] Fix URL input flow (better validation feedback, source detection)
- [ ] Tag system UI (add/remove tags on tracks)
- [ ] Track detail panel (click track → see/edit full info, notes, tags)
- [ ] Filter bar (clickable column headers, filter by source/favorite/tag)
- [ ] Bulk import (drag and drop folder of local files)
- [ ] Duplicate detector UI

## Phase 2 — Metadata & Search
- [ ] YouTube Data API v3 (search YouTube, pull metadata + thumbnail)
- [ ] Spotify Web API (search Spotify, pull metadata)
- [ ] SoundCloud public API (search, streamable tracks)
- [ ] In-app search across all sources

## Phase 3 — Download & Playback
- [ ] yt-dlp integration (download audio from YouTube and other supported sites)
- [ ] Local file playback (NAudio or LibVLCSharp)
- [ ] Mini player / Now Playing bar (progress, volume, play/pause, skip)
- [ ] SoundCloud streaming
- [ ] Keyboard shortcuts (space = play/pause, arrows = navigate)

## Phase 4 — Polish & Power Features
- [ ] Smart playlists (auto-generated by rules)
- [ ] Statistics page (most played, sources breakdown, tracks over time)
- [ ] Listen Later queue (separate from main library)
- [ ] Track journal/notes UI
- [ ] Export to M3U (opens in VLC, mpv etc.)
- [ ] Dark/light theme toggle
- [ ] Keyboard shortcut map

## Phase 5 — Persistence
- [ ] SQLite + EF Core (replace in-memory storage)

---

## Tech Stack
- .NET 8, C#, Avalonia UI (MVVM)
- Serilog (logging)
- xUnit (testing)
- yt-dlp (download, Phase 3)
- NAudio or LibVLCSharp (playback, Phase 3)
- SQLite + EF Core (Phase 5)
- Fedora Linux, VS Code
